# P1 System Scalpels Copy v1.3.1

## mission_cash
title: "Mission ↔ cash"
scalpelQuestion: "Où est écrit l’arbitrage officiel quand mission et cash se contredisent ?"
whyItChangesEverything: "Sans règle, l’arbitrage devient politique : non-dits, coalitions, conflits."
firstCheck: "Peux-tu pointer un document (même 1 page) qui dit ‘si X, alors Y’, validé par une instance ?"
counterHypothesis: "Si c’est écrit mais pas appliqué, le problème est la discipline de gouvernance, pas le texte."

## gouvernance_decision
title: "Gouvernance & veto"
scalpelQuestion: "Qui a le pouvoir explicite de dire NON à un projet ‘finançable mais incohérent’ ?"
whyItChangesEverything: "Sans veto clair, le désaccord se déplace en coulisses."
firstCheck: "Sur une décision récente : peux-tu retracer ‘qui a décidé’, ‘selon quelle règle’, ‘où c’est consigné’ ?"
counterHypothesis: "Si tout est traçable mais contesté, le sujet est la légitimité (mandat/recours), pas la traçabilité."

## dependance_mortelle
title: "Dépendance mortelle"
scalpelQuestion: "Quel est votre point de dépendance mortel (financeur, personne, presta, outil) — celui qui, s’il disparaît, casse la structure ?"
whyItChangesEverything: "La dépendance crée la peur, puis l’opacité."
firstCheck: "Liste 3 dépendances + un plan B réel. Sans plan B, c’est une dépendance mortelle."
counterHypothesis: "Si le plan B existe mais n’ose pas s’activer, le problème est le pouvoir de décider le switch."

## capacite_reelle
title: "Capacité réelle"
scalpelQuestion: "Avez-vous un indicateur qui mesure la capacité réelle (énergie, charge, turnover) — pas juste la charge de travail ?"
whyItChangesEverything: "Sans capacité mesurée, on confond engagement et épuisement."
firstCheck: "Avez-vous une règle simple : ‘au-dessus de 100% → on coupe / on décale’ ?"
counterHypothesis: "Si l’indicateur existe mais n’a aucun effet, le problème est l’arbitrage (qui accepte de couper)."

## pilotage_reporting
title: "Pilotage & reporting"
scalpelQuestion: "Qu’est-ce qui est piloté ici : la mission (impact), ou l’activité (faire tourner) ?"
whyItChangesEverything: "Si on pilote l’activité, on optimise le bruit, pas le sens."
firstCheck: "Quel est le top 3 des métriques qui déclenchent une décision ?"
counterHypothesis: "Si les bonnes métriques existent mais sont ignorées, le problème est la gouvernance (priorités non assumées)."

## controles_internes
title: "Contrôles & sécurité"
scalpelQuestion: "Si j’étais un escroc : quelle est la validation la plus facile à contourner chez vous ?"
whyItChangesEverything: "Quand les contrôles sont faibles, la confiance se transforme en soupçon."
firstCheck: "Sur une dépense/contrat : combien de personnes doivent valider, et qui peut valider seul ?"
counterHypothesis: "Si les contrôles sont solides mais vécus comme arbitraires, le problème est l’équité des règles / la transparence."
